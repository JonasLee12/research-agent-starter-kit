# General Research Skill Compatibility Contract

Purpose: define when a locally developed skill is suitable for this public research-agent starter kit.

## Compatibility Rules

A general research skill should:

1. Avoid hardcoded project paths, personal details, university-specific rules, and single-topic assumptions.
2. State where project-specific source, privacy, citation, compliance, and delivery gates apply.
3. Treat external tools as optional routes, not mandatory installed dependencies.
4. Avoid copying third-party code into the skill unless licence, attribution, and maintenance status are explicitly reviewed.
5. Use neutral `research-*` naming when the skill is intended to transfer beyond one project.
6. Preserve evidence boundaries: retrieval, visual generation, and style improvement do not prove source support.

## Starter-Kit Export Checklist

Before exporting a local skill into this public starter kit:

- remove private project paths and examples;
- replace project-specific gates with generic placeholders;
- check licence and attribution for any referenced third-party tools;
- state required optional tools and install boundaries;
- run a privacy/public-release audit.

## Boundary

This contract supports portability. It does not make a skill safe for every project automatically. Users must still adapt source, privacy, compliance, citation, and delivery gates to their own research context.
