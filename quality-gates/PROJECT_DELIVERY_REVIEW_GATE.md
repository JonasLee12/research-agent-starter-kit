# Project Delivery Review Gate

Use this gate before delivering a formal or high-stakes research output.

It is project-type neutral. Use the more specific rubric, journal, funder, client, ethics, or institutional gate when one exists.

## Project Context

- Project profile: `TO CONFIRM`
- Intended audience: `TO CONFIRM`
- Output type: `TO CONFIRM`
- Requirement source checked: `TO CONFIRM`

## Check Before Delivery

| Check | Question | Status |
|---|---|---|
| Purpose | Does the output answer the user's actual task? | `TO CHECK` |
| Audience | Is the level right for the intended reader? | `TO CHECK` |
| Source grounding | Are factual claims tied to checked sources? | `TO CHECK` |
| Evidence boundary | Are metadata-only sources kept separate from reviewed evidence? | `TO CHECK` |
| Argument | Is the main line of reasoning visible and defensible? | `TO CHECK` |
| Method / process | Does the method or workflow match the claim being made? | `TO CHECK` |
| Citations | Are citations present, matched to references, and claim-support ready where needed? | `TO CHECK` |
| Compliance | Are ethics, privacy, IP, AI-use, or institutional requirements checked? | `TO CHECK` |
| Format | Does the output meet the requested format? | `TO CHECK` |
| Open issues | Are unresolved facts marked `TO CONFIRM`? | `TO CHECK` |

## Output Label

Choose one:

- `working draft`
- `review draft`
- `submission/client-ready after user confirmation`
- `blocked: missing required source or confirmation`

Do not call a document ready when source, compliance, format, or citation evidence is missing.
