# Argumentation Reasoning Framework

Use this reference when a research output needs a defensible argument.

## Claim Status

| Status | Meaning | Action |
|---|---|---|
| Confirmed | Source or project evidence directly supports it | cite or record evidence path |
| Plausible but unverified | The claim is likely but not source-confirmed | mark `TO CONFIRM` or narrow wording |
| Interpretive | The claim is the agent/user's synthesis | label it as interpretation and show warrant |
| Unsupported | No source or reasoning supports it | remove or return to source search |

## Claim-Evidence-Warrant-Boundary Template

```text
Claim:
Evidence:
Warrant:
Boundary:
Source status:
```

## Warrant Quality Questions

1. Evidence fit: does the evidence support this exact claim?
2. Inference step: what reasoning moves from evidence to claim?
3. Alternative explanation: what else could the evidence mean?
4. Scope: where does this claim stop applying?
5. Citation need: does the reader need a source, quote, page, dataset, or policy link?
6. Risk: would this claim be challenged by a supervisor, reviewer, client, funder, or ethics/compliance reviewer?

## Acceptable Gap / Problem Categories

- Conceptual gap
- Empirical/context gap
- Methodological gap
- Implementation/adoption boundary
- Ethics/governance gap
- Policy/practice gap
- Evidence synthesis gap
- Technical/design gap

The category must do work. It should explain why the project needs this study, review, design, or decision.

## Common Failure Patterns

- Topic statement presented as a gap.
- Broad field gap that is already well researched.
- Tool-building gap when the method only supports requirements or perception work.
- Literature summary without an argument.
- Evidence list without a warrant.
- Strong claim built from metadata-only search results.
- Project requirement inferred from memory rather than a source file.

## Revision Moves

If the warrant is weak:

1. narrow the claim;
2. replace a broad gap with a specific gap/problem type;
3. add a source path or mark `TO CONFIRM`;
4. split one overfull paragraph into separate argument jobs;
5. move background material out of the paragraph if it does not support the claim.
