# Research Rhetorical Moves

Use this reference to make section plans specific. A rhetorical plan must name the section type.

## Introduction / Rationale

1. Establish the practical or intellectual issue.
2. Locate the relevant conversation or evidence area.
3. Name the gap/problem type.
4. Explain why the gap matters for this project context.
5. State what this project will examine, design, test, review, or decide.
6. Bound the claim so the project does not promise more than its method can support.

## Literature Review

1. Organise sources by argument role, not by one-source-at-a-time summary.
2. Define the main concepts.
3. Compare what different source groups say.
4. Identify what is already known.
5. Identify what remains uncertain, under-specified, contested, or context-dependent.
6. Show how the review leads to the research question, design choice, or analytical frame.

## Methodology / Methods

1. Restate the research aim or decision need.
2. Justify the method as suitable for the question.
3. Explain participants, data, materials, procedure, or corpus.
4. Explain analysis strategy.
5. Address ethics, privacy, consent, data-management, risk, and compliance where relevant.
6. State limitations and what the method cannot prove.

## Findings / Results

1. State the finding or result.
2. Provide evidence.
3. Interpret cautiously.
4. Distinguish patterns from isolated examples.
5. Connect back to the research question.
6. Avoid turning speculation into finding.

## Discussion / Implications

1. State the main contribution or implication.
2. Explain how it relates to previous work or project requirements.
3. Show practical, theoretical, methodological, policy, or design significance.
4. State limits of transferability or generalisability.
5. Identify next actions, future research, or implementation conditions.

## Proposal / Grant / Report Recommendation

1. State the problem or opportunity.
2. Explain why current evidence or practice is insufficient.
3. Present the proposed work or recommendation.
4. Explain feasibility and fit.
5. Identify risks, dependencies, and compliance requirements.
6. State the decision or support needed from the reader.
