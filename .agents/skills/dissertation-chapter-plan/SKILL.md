# Dissertation Chapter Plan

Use when planning chapters, restructuring drafts, mapping arguments, or turning findings into dissertation sections.

For major proposal or chapter argument logic, use `dissertation-argument-spine` first; use this skill to turn the agreed argument into chapter structure, section plans, and writing schedules.

## Default Chapter Model

Adapt as needed:

1. Introduction
2. Literature Review
3. Methodology
4. Findings
5. Discussion
6. Conclusion and Design Implications

## Workflow

1. Read `DISSERTATION_BRIEF.md`.
2. Inspect existing drafts or notes if present.
3. Identify the intended argument of each chapter.
4. Map research questions to chapters and evidence.
5. Define section-level claims and required evidence.
6. Flag gaps:
   - missing literature
   - missing method detail
   - unsupported findings
   - unclear contribution
   - duplicated discussion

## Outputs

Create or update:

- `chapter-plans/CHAPTER_MAP.md`
- `chapter-plans/ARGUMENT_MAP.md`
- `chapter-plans/WRITING_PLAN.md`

## Guardrails

Keep chapter plans dissertation-oriented, not journal-paper-oriented. Do not compress methodology or findings too aggressively.
