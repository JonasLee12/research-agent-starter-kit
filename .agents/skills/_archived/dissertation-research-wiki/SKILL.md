# Dissertation Research Wiki

Use when creating project memory, organizing notes, tracking decisions, or summarizing progress.

Use `dissertation-knowledge-ops` instead when the task involves cross-store sync, deduplication, source-of-record decisions, or Obsidian/knowledge-base indexing.

## Files

Maintain these files when useful:

- `research-wiki/PROJECT_OVERVIEW.md`
- `research-wiki/LITERATURE_MAP.md`
- `research-wiki/METHOD_DECISIONS.md`
- `research-wiki/DESIGN_RATIONALE.md`
- `research-wiki/FINDINGS_INDEX.md`
- `research-wiki/SUPERVISOR_FEEDBACK.md`
- `research-wiki/OPEN_QUESTIONS.md`

## Workflow

1. Read `DISSERTATION_BRIEF.md`.
2. Inspect existing wiki files before adding new notes.
3. Record decisions with date, rationale, evidence, and open risks.
4. Keep summaries short and traceable to source files.
5. Do not store raw sensitive participant data in wiki files unless anonymized.

## Output Style

Use concise headings, dated entries, and action lists.
