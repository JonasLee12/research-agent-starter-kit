# Prototype Evaluation Audit

Use when reviewing a prototype, demo, wireframe, agent prompt, interaction flow, user test, or design concept.

## Evaluation Dimensions

Check:

- usability for university teachers
- support for active learning implementation
- teacher control and autonomy
- clarity of agent outputs
- fit with preparation and classroom workflow
- adaptability to discipline and class size
- handling of uncertainty and bad advice
- privacy and responsible AI safeguards
- evidence quality from teacher feedback
- alignment with dissertation claims

## Workflow

1. Identify what is being evaluated:
   - conceptual prototype
   - clickable prototype
   - prompt-only agent
   - working software
   - scenario storyboard
2. Read available evaluation evidence.
3. Separate:
   - observed usability issues
   - participant preferences
   - researcher interpretation
   - design recommendations
4. Map each issue to severity:
   - high: threatens adoption, ethics, or core teaching use
   - medium: weakens usability or clarity
   - low: polish or wording
5. Recommend next iteration steps.

## Output

Create or update:

- `audit-reports/PROTOTYPE_EVALUATION_AUDIT.md`
- `design-specs/PROTOTYPE_REVISION_PLAN.md`

## Guardrails

Do not claim validated effectiveness from informal feedback.
Mark whether evidence comes from actual teacher use, speculative review, or researcher judgment.
