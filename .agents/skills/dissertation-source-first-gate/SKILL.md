# Dissertation Source-First Gate

Use this skill before drafting, revising, or auditing formal project materials where factual accuracy matters.

## Purpose

Prevent invented administrative facts, personal details, participant information, citations, dates, deadlines, supervisor comments, and university requirements. This adapts the ECC GateGuard idea into a dissertation-specific manual gate.

## Trigger This Gate Before

- ethics forms
- participant information sheets
- consent forms
- recruitment emails
- interview guides and appendices
- proposal drafts
- supervisor-facing documents
- LMS/module requirement summaries
- marking criteria, rubrics, 75+ standards, grade bands, deadlines, word counts, or submission rules
- citation-heavy literature text
- personal identity or administrative information updates

## Required Source Map

Before drafting or editing, identify:

1. user request
2. source files / URLs / screenshots / LMS pages used
3. confirmed facts
4. fields that are `TO CONFIRM`
5. assumptions or interpretations
6. information that must not be invented
7. intended files to create or modify

For rubric, 75+, grade-band, LMS requirement, deadline, word-count, or submission-rule questions, also identify:

8. whether the evidence is `OFFICIAL ORIGINAL TEXT`, `LOCAL SUMMARY`, `INFERENCE FROM LOCAL SUMMARY`, or `EVIDENCE INSUFFICIENT`
9. whether exact official wording is available locally
10. whether `university-guidance/RUBRIC_EVIDENCE_GATE.md` has been checked

For school templates:

- inspect the template before editing
- preserve fixed black template text unless clearly editable
- replace only editable placeholder or guidance text
- preserve source-confirmed names, emails, dates, and supervisor details

## Rubric / LMS Requirement Gate

Use `university-guidance/RUBRIC_EVIDENCE_GATE.md` before answering or drafting from marking criteria, 75+, grade-band, LMS, deadline, word-count, or submission-rule claims.

Minimum files to check first:

- `university-guidance/RUBRIC_EVIDENCE_GATE.md`
- `university-guidance/RUBRIC_OR_MARKING_CRITERIA_TEMPLATE.md or the project-specific rubric file`
- `university-guidance/MODULE_REQUIREMENTS_TEMPLATE.md or the project-specific module-requirements file`
- `university-guidance/FORMAT_REQUIREMENTS_TEMPLATE.md or the project-specific format file`

Do not present a local summary as the full official rubric. If exact wording is unavailable, say so and mark the exact wording as `TO CONFIRM`.

## Current Administrative Fact Placeholders

Use only when relevant and still check the active source file if the target document is a formal form.

- Default prose name: `TO CONFIRM`
- Administrative/source-record name: `TO CONFIRM`
- Email: `TO CONFIRM`
- Programme: `TO CONFIRM`
- Supervisor: `TO CONFIRM`
- School/department: `TO CONFIRM`
- Personal or researcher status: `TO CONFIRM`, internal context only unless the user explicitly says to disclose it

## Citation Gate

For literature claims:

- verify source metadata
- check whether the source actually supports the claim
- mark unverified citations as `NEEDS VERIFICATION`
- do not invent page numbers or quotations
- use `CONTEXTUAL SOURCE` for non-peer-reviewed thinking resources

## Output Contract

For substantial work, include a short source map before or inside the working note:

```text
Source map:
- Used:
- Confirmed:
- TO CONFIRM:
- Inference:
- Files to update:
```

For short chat answers, apply the gate silently unless an uncertainty affects the answer.

## Stop Conditions

Stop and ask or mark `TO CONFIRM` when:

- a required name, date, email, deadline, module rule, or ethical condition is missing
- a required rubric, grade-band, word-count, LMS requirement, or submission rule is not available as source text
- two source files conflict
- a participant-facing disclosure is not clearly authorised
- a formal university template is ambiguous about editable text
- a citation cannot be verified enough for the claim
