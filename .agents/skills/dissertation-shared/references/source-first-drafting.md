# Source-First Drafting Protocol

Use this protocol before drafting or revising formal dissertation, ethics, proposal, LMS, or university-template documents.

## Core Rule

Do not invent factual fields. Extract first, then draft.

Factual fields include:

- student name
- student email
- supervisor name
- supervisor email
- school, programme, module, and institution names
- dates, deadlines, signatures, approval status, and submission requirements
- participant counts, recruitment route, and data collection details
- storage, retention, transcription, and AI-tool arrangements

If a field is not present in the source material, write `TO CONFIRM` rather than guessing.

## Required Workflow

1. Identify the source file(s) that may already contain the facts.
2. Extract existing filled fields before writing.
3. Create a short source map when the task involves forms or official requirements.
4. Separate:
   - `CONFIRMED FROM SOURCE`
   - `DRAFTED FROM PROJECT CONTEXT`
   - `TO CONFIRM`
5. Use confirmed fields in the draft.
6. Ask the user only for fields that cannot be found and are needed for the next step.

## University Template Rules

When editing a school or university template:

- inspect template structure before editing
- preserve fixed template text and formatting
- identify editable placeholders, examples, and optional instructions
- do not overwrite whole template sections with free-form prose unless the user explicitly asks
- if colored text is used, inspect what the colors mean before changing content
- remove unused optional instructions before final submission

For the school/department ethics form:

- black text is usually template wording or fixed structure
- blue `xxx` text is usually a placeholder to replace
- blue bracketed text is usually guidance to adapt or delete
- optional bracketed consent/data-sharing wording should only remain if it applies to the study
