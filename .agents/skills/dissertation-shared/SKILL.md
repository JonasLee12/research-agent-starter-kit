# Dissertation Shared Protocols

Use this skill as a reference layer for all dissertation tasks.

Read only the relevant reference file when needed:

- `references/privacy-and-ethics.md`: participant data, anonymization, consent, and sensitive content.
- `references/citation-discipline.md`: citation verification and anti-hallucination rules.
- `references/review-protocol.md`: independent review behavior and evidence boundaries.
- `references/output-contracts.md`: recommended local files and revision logs.
- `references/source-first-drafting.md`: source-field extraction, personal information safeguards, and university template editing rules.

Core rules:

1. Separate evidence, interpretation, and recommendation.
2. Never invent citations, quotes, participants, findings, or supervisor comments.
3. Never invent personal details, university contacts, supervisor names, emails, dates, or administrative requirements.
4. Prefer anonymized summaries over raw participant data.
5. Use exact local file paths when referring to evidence.
6. For revisions, state what changed and why.
7. Use `dissertation-source-first-gate` before formal drafting or template editing.
8. Use `dissertation-document-quality-gate` before delivering formal or supervisor-facing outputs.
